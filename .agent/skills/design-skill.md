# Design Skill

## 核心目标

把界面设计成科技、艺术、生命力与稀缺性交汇的表达，而不是安全、平庸、无记忆点的中性方案。

最终输出应让人感到被点燃，而不是被压迫或被焦虑驱动。

## 精神内核

- 传达新时代的火种、车库精神、Builder 心态。
- 保留冒险感、速度感、前进方向和做难事的勇气。
- 允许原型感、未完成感和功能主义痕迹存在，但整体必须有控制力。
- 强调希望感，而不是冰冷、权威、金融化的压迫感。

## 系统一致性

- 统一色调、字体、间距、圆角、按钮、卡片、标签和图形语言。
- 所有页面共享同一套视觉语法，不做每个区块各说各话。
- 先定义设计变量，再扩展组件，不要先堆局部技巧。

## 有效性原则

- 每一个元素都必须服务于识别、层级、情绪或转化目标。
- 新增一个效果前，先判断用户能否感知，以及它是否真的让目标更强。
- 如果某个技巧只是“看起来设计过”，但不提升结果，删除。
- 优先少而准，不靠堆砌制造高级感。

## 色彩系统

- `Deep Navy #060644`：品牌底层色，承担标题、主品牌识别、Logo 主体。
- `Electric Blue #0909E2`：高能强调色，小范围谨慎使用，用于关键动作和焦点。
- `Pulse Aqua #27D7C7`：次亮色，用于局部点燃、记忆点和能量感。
- `Foam White #F6F7FB`：主浅底色，比纯白更柔和，更适合长时间阅读。
- `Slate Gray #F5F5F5`：低优先级背景或过渡层，不要用成主品牌色。

使用要求：

- 用主色构建色阶，而不是随意补色。
- 高亮色只在焦点处出现，避免满屏发光。
- 不要退化成廉价蓝紫渐变赛博风。

## 视觉元素

- 海浪：用曲线、流动、层叠和方向性传达节奏与推进。
- 火种：用局部高亮、微小能量点和点燃式对比制造精神性唤醒。
- 发射：用上升、速度、轨迹、环绕和冲刺感建立冒险叙事。
- Demo 精神：保留少量粗粝、原型感和未完成感，但必须可控。
- 真实：优先真实照片、真实环境、真实动作、真实记录；少用假大空素材。
- 极简：用留白、呼吸感和克制排版，而不是信息稀薄。

## 排版原则

- 标题要短、狠、清晰，有方向感。
- 正文只保留必要信息，避免海报式大段说明。
- 信息层级必须一眼可读，不用依赖复杂装饰解释重点。
- 排版要有呼吸感，但不能空洞。

## 组件原则

- 按钮要明确区分主次动作。
- 卡片优先承载结构化信息，不只做视觉容器。
- 标签要承担分类和节奏作用，不要沦为装饰。
- 动效以轻量、方向性、状态反馈为主，不炫技。

## 明确禁止

- 不做极繁主义和装饰堆砌。
- 不做正式严肃的黑金金融媒体风。
- 不做过量文案海报。
- 不做廉价赛博风，不依赖泛蓝紫渐变、重发光、HUD 元素。
- 不做过度中和后的“安全方案”。

## 执行顺序

1. 先判断页面要传达的核心精神，是火种、冒险、真实还是秩序。
2. 先建立色彩、排版、留白和组件变量，再设计单个区块。
3. 先做最强识别区，如 Hero、导航、主按钮、主卡片。
4. 再把同一套视觉语言扩展到列表、详情、侧栏、空状态和表单。
5. 最后删掉无效技巧，确认页面是否仍然有锋芒、有识别度、有一致性。

## 自检问题

- 这个页面是否有明确记忆点？
- 是否传达了 Builder 精神，而不是模板站气质？
- 焦点是否足够集中？
- 高亮是否克制？
- 留白是否有呼吸感，而不是空？
- 如果删掉一半装饰，核心气质是否还成立？
